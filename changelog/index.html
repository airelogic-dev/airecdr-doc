    Changelog

    Welcome to the Airelogic controlled drugs register changelog. Here you can find out more about recent Aire CDR releases and what the changes mean for you as a Aire CDR user.

    March 3, 2023

    Version 1.4.9 General improvements

    • Prevent setting a PIN without a user ID
    • Additional logging around JSON web tokens
    • Rework of stock count to make code more efficient
    March 3, 2023

    Version 1.4.8 Changes following support calculation

    • Improve location handling
    • Enable consistent reads for stock events
    • Store and recall user information directly on/from stock events
    • Add doop jar ID to additional information on administration/broken
    • Improve 'additional information' sidebar user interface
    • Move sign out button inwards from border
    February 28, 2023

    Version 1.4.7 Fix for multiple-checkout bug

    • Update projects to .NET Standard 2.1/.NET 6.0
    • Add name and staff number in plain text to all stock event and note records
    • Ensure that users with the same name cannot have the same PIN set (as this would prevent them logging in)
    February 3, 2023

    Version 1.4.6 Fix for stock count bug

    • Check that schedule 2 drugs are active before asking for stock count
    February 2, 2023

    Version 1.4.5 Fix for multiple-checkout bug

    • Ensured consistent reads across additional pouch functions
    January 30, 2023

    Version 1.4.4 Usability improvements and date indexing fix

    • Fixed bug where stock events before 19th December were not included in by-date search
    • Fixed 'origin' and 'destination' display bug on stock book
    • Put site locations in alphabetical order
    • Added check on location page to prevent accidental duplication of location names
    • Removed submission options from drug returns page and added 'doop' option to 'record event' list
    January 9, 2023

    Version 1.4.3 Bug fix following support request

    • Disabled 'Collect Returns' submission button while system is processing returns
    • Added more detail to system logs
    • Removed border from underneath 'saving' indicator
    December 19, 2022

    Version 1.4.2 Bug fix following Willenhall go-live

    • Fixed issue where system error prevented deliveries from being received
    November 30, 2022

    Version 1.4.1 Additional feature to ensure legal compliance

    • Allowed users to select a 'Deliveries' location listing stock changes on and off of delivery vans
    November 23, 2022

    Version 1.4.0 Bug fixes following Warwick go-live

    • Allowed users to specify that no witnesses are available for confirming following a stock count discrepancy
    • Cleansed (trimmed whitespace from) key strings including users' names and email addresses
    • Ensured that names and IDs of new locations are cleansed before being stored and used
    November 18, 2022

    Version 1.3.5 Workaround for common problem

    • Added warning to users on checkout to show how many and which pouches (if any) the user has assigned to them.
    November 10, 2022

    Version 1.3.4 Bug fix following feedback from client

    • Ensured that pouches are read from the database using strong consistency to fix 'pouch lag' problem
    October 25, 2022

    Version 1.3.3 Bug fix following feedback from client

    • Fixed a bug where manual doops were incorrectly adjusting the stock count
    October 17, 2022

    Version 1.3.2 Changes and bug fixes following feedback from client

    • Changed the way broken ampoules are handled. Allowed recording of amounts dooped / lost and ensured stock is adjusted accordingly
    September 30, 2022

    Version 1.3.1 Changes and bug fixes following feedback from client

    • Improved test coverage
    • Ensured out of date drugs always show on returns page
    • Fixed count discrepancy when unable to replenish pouches
    • Sanitise signature text box contents
    September 28, 2022

    Version 1.3.0 Changes and bug fixes following feedback from client

    • Changing DOOP counts so that multiple ampoules are counted as one jar on return.
    • Bugfix for count on multiple administrations/replenishments across multiple pouches for the same drug
    September 13, 2022

    Version 1.2.0 Changes and bug fixes following feedback from client

    • Ensured stock counts are listed in descending date order (most recent at the top)
    • Removed empty stock counts from stock count list
    • Ensured that morphine counts are required on check out and check in, but are not (unless used) on replenishment.
    • Changed stock book so that out of date drugs remain in overall stock count but are recorded as out of date
    • Fixed stock calculation when multiple vials are used
    • Fixed bug where drug returns are missing from the Drug Returns page
    August 24, 2022

    Version 1.1.4 Changes and bug fixes following feedback from client

    • Added support for declaring number of ampoules used and dooped during administration
    • Added configuration setting to prevent OIDC sign-in if PIN sign-in is available
    • Ensured that a count of schedule 2 drugs is required after replenishment if a schedule 2 drug has been used and is always required on check in
    • Removed 'This site is currently under development' notice from copyright footer
    August 5, 2022

    Version 1.1.3 Bug fix following feedback from client

    • Fixed bug to prevent system crash when an empty user record is created
    August 3, 2022

    Version 1.1.2 Change following feedback from client

    • Added count for schedule 2 drugs on check in as well as check out
    August 2, 2022

    Version 1.1.1 Changes and bug fixes following feedback from client

    • Changed stock book text colour for discrepancies from red to black
    • Fixed bug where Dispatch permission was unavailable
    July 7, 2022

    Version 1.1.0 Additional features to ensure legal compliance

    • Allowed global admins to set location on new set up
    • Changed logo
    • Allowed users to select multiple roles
    • Added count of logged in users
    • Improved layout
    • Allowed decimal amounts (instead of whole numbers)
    • Allowed administration of "0" amount of drug
    • Required confirmation if user clicks 'Start Again' after replenishing
    • Added ability to record notes and incident number during stock counts
    • Changed font colour of stock counts in books to red
    • Require stock count of replenished / checked-in drugs
    • Added ability to transfer control of pouches
    • Ensured correct pouch is selected when changing stock if two pouches of the same type are checked out
    • Allowed recording of administrations when not able to replenish pouches
    • Added witnessing of unexpected stock counts during replenishment / check-in
    • Added coloured border around site when not on production
    Sept 23, 2021

    Version 1.0.0 Optimisations and improvements.

    • Allowed leading zeros on PIN entry pad
    • Added expiry time for session when using PIN login
    • Added ability to edit a pouch without changing its name
    • Added prompt for user to sign out after some actions
    • Improved colour contrast and size of text / interactive elements
    • Configured numeric inputs to use a numeric keyboard
    • Improved PIN management user interface
    • Added UI hints
    • Added ability to search users page
    Sept 2, 2021

    Version 0.0.5 Additional features

    • System now automatically adds broken/dooped items to returns on check-in
    • Implemented mutual TLS for PIN sign-in
    • Added tenant-based files for IDP configuration
    • Doop jars can now be recorded as received
    • Added ability to mark return type as destroyed
    • Added ability to delete roles
    • Changed roles UI to dropdowns instead of text entry
    • Fixed note formatting
    • Stock count is now requested when replenishing drugs
    • Added ability to rename an existing role
    • Made sign out button more accessible
    • Added ability to create a user without an invitation
    Jul 26, 2021

    Version 0.0.4 Bug fixes and improvements

    • Fixed issue with case sensitivity for drug IDs
    • Users can record a supplier against a delivery date
    • Allows users to enter name in lowercase for pin sign in
    • Users can edit drug colours for drug book view
    • Roles and permissions are now saved correctly on the user management screen
    • Fixed issues caused by wrong permissions being assigned to certain actions
    • Add signatures on collection of drug returns
    • Improved UI on pouches page
    • Made sign in icons clickable
    • Prevent user from finishing and checking out before completing required steps
    • Broken drugs are returned to stock on check in
    • Improved usability of numeric input boxes
    • New feature to check in and destroy drugs at central hub
    • Corrected stock counts during check in and check out
    • Added progress indicators throughout site
    Jul 05, 2021

    Version 0.0.3 Pouch types, book colours and drug returns

    • Introduced pouch types and a new way for managing pouches
    • Pouches are now tracked across location so OMs can see which pouches have moved
    • Book view now displays drug specific colours to make identification easier
    • Drugs can now be marked for return to the stock hub
    • Weekly delivery schedule information can now be viewed and managed
    • Fridge temps are recorded as part of a stock check
    • PIN based log in is now available
    Jun 11, 2021

    Version 0.0.2 User management and more improvements

    • Implemented a new home page layout
    • Added ability to remove recipients in notifications management
    • Stock discrepancies are now highlighted in the drug book view
    • Added basic user management functionality
    • Other drug schedule now available in drugs management
    • Origin and destination of stock events now shown in book view
    • Add ability to edit location names
    Jun 01, 2021

    Version 0.0.1 General bug fixes and improvements

    • Added paging to the stock requests list to view older requests.
    • Added ability to clear the user filter selection on the drug events page.
    • Fixed some spelling and display problems.
    • Improved system handling of checking out multiple pouches.
    • Removed the ability to dispatch drugs for a stock request from the same location it was created.
    • Improved the UI of warnings in the system.
    • Redesigned the home page layout.
    • Ordered the locations alphabetically in the notifications management page.
    • Added a page to notify the user when there has been an error in the system.
    • Improved the logging of system errors to aid investigation.
    • Improved retry attempts in larger database operations to reduce errors.
